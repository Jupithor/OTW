Welcome to OverTheWire!

If you find any problems, please report them to Steven or morla on
irc.overthewire.org.

--[ Playing the games ]--

  This machine might hold several wargames.
  If you are playing "somegame", then:

    * USERNAMES are somegame0, somegame1, ...
    * Most LEVELS are stored in /somegame/.
    * PASSWORDS for each level are stored in /etc/somegame_pass/.

  Write-access to homedirectories is disabled. It is advised to create a
  working directory with a hard-to-guess name in /tmp/.  You can use the
  command "mktemp -d" in order to generate a random and hard to guess
  directory in /tmp/.  Read-access to both /tmp/ and /proc/ is disabled
  so that users can not snoop on eachother. Files and directories with
  easily guessable or short names will be periodically deleted!

  Please play nice:

    * don't leave orphan processes running
    * don't leave exploit-files laying around
    * don't annoy other players
    * don't post passwords or spoilers
    * again, DONT POST SPOILERS!
      This includes writeups of your solution on your blog or website!

--[ Tips ]--

  This machine has a 64bit processor and many security-features enabled
  by default, although ASLR has been switched off.  The following
  compiler flags might be interesting:

    -m32                    compile for 32bit
    -fno-stack-protector    disable ProPolice
    -Wl,-z,norelro          disable relro

  In addition, the execstack tool can be used to flag the stack as
  executable on ELF binaries.

  Finally, network-access is limited for most levels by a local
  firewall.

[1mDESCRIPTION[0m[m
       OpenSSL is a cryptography toolkit implementing the Secure Sockets Layer (SSL v2/v3)[m
       and Transport Layer Security (TLS v1) network protocols and related cryptography[m
       standards required by them.[m
[m
       The [1mopenssl[0m program is a command line tool for using the various cryptography[m
       functions of OpenSSL's [1mcrypto[0m library from the shell.  It can be used for[m
[m
        o  Creation and management of private keys, public keys and parameters[m
        o  Public key cryptographic operations[m
        o  Creation of X.509 certificates, CSRs and CRLs[m
        o  Calculation of Message Digests[m
        o  Encryption and Decryption with Ciphers[m
        o  SSL/TLS Client and Server Tests[m
        o  Handling of S/MIME signed or encrypted mail[m
        o  Time Stamp requests, generation and verification[m
[m
[1mCOMMAND[0m [1mSUMMARY[0m[m

       above), each of which often has a wealth of options and arguments ([4mcommand_opts[24m and[m
       [4mcommand_args[24m in the SYNOPSIS).[m
[m
       Many commands use an external configuration file for some or all of their arguments[m
       and have a [1m-config[0m option to specify that file.  The environment variable[m
       [1mOPENSSL_CONF[0m can be used to specify the location of the file.  If the environment[m
       variable is not specified, then the file is named [1mopenssl.cnf[0m in the default[m
       certificate storage area, whose value depends on the configuration flags specified[m
       when the OpenSSL was built.[m
[m
       The list parameters [1mstandard-commands[0m, [1mdigest-commands[0m, and [1mcipher-commands[0m output a[m

       [1mopenssl[0m utility.[m
